In this project, we developed a basic rasterization engine capable of rendering lines, triangles, and shapes from SVG files, adding color and texture. We explored various sampling techniques for texture filtering and mipmap level adjustment.

The experience enhanced our understanding of applying theoretical formulas to practical image generation and rendering, translating class-learned concepts into visual outcomes based on coordinates and directives.

To rasterize triangles, we will first create a bounding box for the triangle. This is achieved by calculating the minimum and maximum x and y coordinates among the vertices and rounding them to integer values to establish the boundaries of the loop.

Then, for each integer coordinate within this bounding box, we check whether each pixel lies inside the triangle by using the three-line test. Essentially, this works by treating each line as the boundary of a half-plane, where the half-plane includes the area within the triangle. If a point is inside all three half-planes, then it must be inside of the triangle.

If a pixel is determined to be inside the triangle, the fill_pixel function is called to color it with the specified color. This process repeats for every pixel within the bounding box, effectively rasterizing the triangle.

Overall, the algorithm's integer grid-based approach, coupled with geometric principles, minimizes unnecessary computations, ensuring that it performs no worse than one that checks each sample within the bounding box. In fact, it likely outperforms such a brute-force approach in terms of both computational efficiency and accuracy in rasterizing the triangle.

  1. Walkthrough of Supersampling Algorithm and Data Structures:

    In supersampling, we enhance image quality by subdividing each pixel into smaller segments and sampling from each segment. This division helps mitigate high-frequency artifacts and jagged edges.

    To implement this, we add two nested loops within each pixel to iterate through subsamples, repeating the process sample_rate times. Within these loops, we still utilize the method to check if the subsample point falls within the triangle, writing the result to the sample_buffer vector. This buffer stores the color values for each subsample point within a pixel. To generate the final image, we use the resolve_to_framebuffer method, which computes the average color from the subsample points for each pixel, thereby producing a smooth and improved final output.

  1. Why Supersampling is Useful:

    By subdividing pixels into smaller segments and sampling multiple points within each segment, supersampling effectively increases the resolution of the rendered image. This process helps to capture finer details and smoother transitions, resulting in a more visually pleasing and realistic final image.

Barycentric coordinates are a coordinate system utilized for seamless interpolation within triangles. They enable straightforward assessment of proximity to each vertex and maintain accurate ratios between them. This facilitates smooth transitions in color or texture, aiding in anti-aliasing and simplifying point sampling within the triangle.

In texture mapping, pixel sampling involves determining the color of each pixel on a rendered surface by sampling the corresponding texture image. This is achieved by interpolating between texels (texture elements) to find the appropriate color values for each pixel based on its position on the surface and the texture coordinates associated with it.

Within our project, we implemented both nearest and bilinear pixel sampling methods. Nearest sampling selects the color of a pixel based on the nearest texel in the texture, which is simple but can lead to aliasing artifacts due to sharp transitions. Bilinear sampling improves upon this by blending colors from nearby texels using interpolation, resulting in smoother transitions and reduced aliasing.

Bilinear pixel sampling images exhibit smoother edges and fewer artifacts in comparison to images generated using nearest sampling with the same supersampling rate. This contrast is most evident when textures feature distinct color boundaries, as illustrated in the provided example where a sharp cutoff separates blue and yellow regions in the circle. Because of the occur of high frequency change, nearest sampling without supersampling struggles to handle such color transitions, resulting in noticeable jagged edges. By employing bilinear sampling and increasing the supersampling rate, the antialiasing effect is enhanced, leading to significantly smoother images.

Level Sampling refers to the method of selecting the appropriate mipmap level for a given image. This selection process involves calculating the derivative of a single pixel about dx and dy and subsequently applying Barycentric coordinates to transition this into uv space. Through this process, we can assess the degree of stretching or compression present in the image, which in turn guides the determination of the most suitable mipmap level for the image. Our implementation covers mipmap level zero, along with nearest level sampling and bilinear level techniques.

  • L_ZERO: Utilizing only the original texture (Level 0) for the entire image, L_ZERO presents the simplest form of texture sampling. This method minimizes computational demands and memory footprint by avoiding the need for additional textures or levels. The major trade-off, however, is its susceptibility to aliasing artifacts. Due to the absence of any sophisticated antialiasing processes, images processed with L_ZERO may exhibit noticeable jaggedness or pixelation, especially in high-contrast areas or at steep viewing angles.
  • L_NEAREST: By calculating and rounding the appropriate level to the nearest whole number, L_NEAREST samples from a specific mipmap level. This technique represents a middle ground in the spectrum of texture sampling methods. It improves upon L_ZERO by utilizing mipmaps to reduce aliasing, thus offering better image quality at the cost of increased memory usage to store these additional texture levels. The computational overhead is moderate, as it involves simple rounding operations and direct sampling from the determined mipmap level, balancing processing speed with a moderate level of antialiasing.
  • L_LINEAR: The most sophisticated among the three, L_LINEAR interpolates between the two levels closest to the calculated value using bilinear filtering. This technique requires the highest computational resources due to the need for calculating floor and ceiling functions for level selection, followed by performing interpolation between color values from these levels. The benefit of this complex process is a significant enhancement in antialiasing quality, yielding smoother and more visually appealing images. The trade-offs for this high-quality output are increased memory usage for storing multiple levels of mipmaps and the substantial computational effort needed for the interpolation process.
