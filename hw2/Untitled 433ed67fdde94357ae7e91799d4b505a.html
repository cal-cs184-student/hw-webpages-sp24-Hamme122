In this project, we learned and implemented the creation and manipulation of Bezier curves and surfaces, the calculation of area-weighted vertex normals, the efficient mesh manipulation such as edge flipping and splitting in the halfedge data structure, and the application of loop subdivision for mesh refinement. We also learned a lot from the process of debugging.

De Casteljau's algorithm is a recursive approach for evaluating Bezier curves, which recursively linearly interpolates between the control points of the Bézier curve until a single point is obtained. This point is on the Bezier curve for the given parameter t, which varies from 0 to 1.

In our implementation, we finished the evaluateStep function, where for a given set of points, it computes the next set of intermediate points by linearly interpolating between each pair of adjacent points using the parameter ttt﻿. In every call to this function, we establish a loop that iterates through each pair of adjacent points in the input vector, linearly interpolates between them, and stores the result in a new vector to return.

Below we show our result of a bezier curve with 6 control points. We can see each step of the evaluation from the original control points down to the final evaluated point. And we also show the result after moving the original control points around and modifying the parameter ttt﻿.

De Casteljau's algorithm extends to Bezier surfaces by applying the same principle used for Bezier curves but in two dimensions. For a Bezier surface, control points are arranged in a grid (or a mesh) rather than a single sequence.

In our implementation, we finished 3 functions here. This first function BezierPatch::evaluateStep linearly interpolates between original control points to simplify them, creating intermediate points. The second function BezierPatch::evaluate1D loops through the levels of points (the amplification) until we get a single point that represents a specific location on the Bezier curve or surface. And the last function BezierPatch::evaluate loops through the whole mesh, by taking the control point of each curve that we calculated previously and then calculating the singular point between them to create another singular control point that thus works in both the x and y direction.

In the process of implementing the area-weighted vertex normals, we utilized an iterative approach that loops through all the neighboring triangles of a given vertex. For each triangle, we find its three vertexes first and then we calculate the normal by taking the cross product of two edges emanating from the vertex. We also calculate the triangles’ area using half of the cross product. Then each normal is then scaled by the area of the triangle, and we got the final vertex normal by normalizing this sum. We implemented loop subdivision by using five separate for-loops that go through the whole mesh. The first and second for-loops work to find the new positions of each new vertex, which allows for the averaging out or smoothing effect. For the positions of the old vertices, we checked it’s neighbors in a rhombus-like path. For the positions of new vertices, we went through all of the edges to find the closest points. The third loop focuses on splitting each edge into 2 and the fourth for-loop flips the edges appropriately, cleaning up the mesh. The final for-loop then moves each vertex to it’s calculated position in the model.

After the loop subdivision, the sharp corners and edges start to become curves and thus beginning to smooth out. You can probably reduce this effect by having more splits and points along the edges and vertices. This is because the points in the model remain mostly unchanged, instead the faces and edges are formed by averaging these points. As such, the more points there are in an area, the less they are affected by smoothing.

Pre-processing can help alleviate these effects as our current algorithm averages the vertices’ positions while simultaneously doing subdivision. Since our mesh is built diagonally-leaning the smoothing will also follow that shape. However, if we were to divide up our points before-hand we would have more information on how to average out vertex positions, thus maintaining the shape of the cube. In terms of making it more symmetric, we can change the way that we split the edges so that both sides of the mesh are the same, rather than having it lean to one side like before.
